Boas práticas em Excel

Faz tudo como se amanhã acordasses amnésico

  • Evita nomes confusos ou vagos — o nome de cada ficheiro, separador, tabela, gráfico, etc. deve permitir identificar imediatamente o que é que essa coisa trata
  • Assume sempre que podemos voltar aos dados daqui a meses ou anos e não nos vamos recordar de como as coisas estão organizadas se não formos descritivos
  • Sê descritivo como se outra pessoa fosse pegar na base de dados
  • Pode ser útil acrescentar comentários às células ou criar um separador com legenda
  • Dar títulos ou legendas breves aos gráficos e valores, mesmo que seja só para nós!

Mantém apenas uma tabela por folha

  • Regra geral devemos separar os inputs dos outputs — isto é, não devemos fazer tratamento de dados na mesma folha onde esses dados são colhidos / introduzidos
  • Existe risco de ocultarmos dados não relacionados quando aplicamos filtros a uma tabela
  • Manter tabelas dinâmicas, gráficos, etc nas suas próprias folhas
  • Por vezes pode ser útil ter folhas adicionais para legendas, conclusões ou resumos

Não mistures alhos com bugalhos

  • Manter separadores juntos no mesmo ficheiro se fizer sentido, caso contrário movê-los para ficheiros à parte
  • Se há dados que não são relevantes, apaga-os ou oculta-os
  • Mau: dados-2019-todos.xlsx
  • Bom: pasta de "dados" com subpasta "2019" com múltiplos ficheiros (dados de imc, dados de diabéticos, dados de hipertensos, dados da consulta infantil, ...)

Valida os dados antes de os introduzir

  • 1 dado mal preenchido em 1000 é o suficiente para gerar erros de fórmulas que se podem revelar uma dor de cabeça
  • Perder algum tempo a acrescentar validação de dados a priori ou menu "dropdown" pode facilitar muito a introdução de dados válidos

O prático não tem de ser bonito, mas o feio raramente é prático

  • Formata as coisas de forma a facilitar a introdução, validação e tratamento de dados
    • Contudo, não perder tempo com excesssos a não ser que seja para uma apresentação
  • Não deixar colunas com dados truncados ou parcialmente ocultos (usar abreviaturas é OK)
  • Uma tabela com linhas listadas é o suficiente para evitarmos introduzir dados na linha errada, por exemplo
    • Podemos e devemos formatar condicionalmente as células vazias que deviam estar preenchidas

Célula vazia não é célula nula

  • Evitar usar o vazio para significar "não"
  • Se acrescentarmos linhas facilmente perdemos o rasto do que já foi ou não preenchido
  • Alternativas: notação "sim/não", 1/0, cores...
  • Usar notação 1/0 em vez de X

Evita "fórmulas mágicas"

  • As constantes e números "mágicos" devem ser introduzidos primeiro em células destinadas e claramente formatadas para o efeito
    • Assim sabemos exactamente onde estão, para o que servem e permitimos que qualquer pessoa possa alterá-los sem ter de alterar as fórmulas
    • Exemplo: promoção "iva zero"
  • Pelo mesmo motivo, devemos evitar fórmulas que sejam demasiado longas
    • Mais vale manter fórmulas intermédias em colunas à parte, que podem ou não ser ocultas
    • Assim qualquer pessoa que não o autor consegue perceber facilmente o que faz cada fórmula e como alterá-la se necessário

Usa a cloud

  • Backups são úteis, mas não devem ser misturados com os ficheiros actualizados para não corrermos o risco de repetir trabalho ou usar dados desactualizados
  • Dropbox, OneDrive, Google Drive, etc.
  • Ninguém merece perder anos de dados guardados na pen
  • Ninguém merece andar a enviar mails para si próprio

Sê consistente (Stick to the plan)

  • Procurar manter todos os dados relevantes organizados no mesmo sítio
  • Manter a ordem que escolhemos para os separadores (ex: primeiro os separadores de colheita, depois tratamento, depois gráficos, ou vice-versa)
  • Não alterar radicalmente a organização a meio da colheita ou tratamento dos dados, sob pena de gerar incompatibilidades entre bases de dados.

Reconhece que ninguém é perfeito

  • O óptimo é inimigo do bom
  • Ser prático e eficiente implica nem sempre ser perfeito
